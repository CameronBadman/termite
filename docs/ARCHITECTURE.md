# Architecture

Termite is split into a terminal core and an application shell.

## Crates

`c_term_core` owns terminal semantics:

- VTE parsing
- grid state
- cursor state
- scrollback
- terminal modes
- damage tracking
- terminal identity constants and query replies

`c_term_app` owns the operating-system boundary:

- Wayland window creation through `winit`
- PTY spawning and resizing
- shell input/output
- clipboard integration
- render cache
- `wgpu` presentation
- compiled config
- plugins

The boundary is intentionally plain. PTY bytes go into `TerminalCore`, and the
result is a `CoreTick` containing damage, terminal responses, and clipboard
events.

## Event Model

The app uses `winit`'s event loop in wait mode. Frames are requested when work
exists:

- PTY reader receives bytes
- input writes to the PTY
- resize changes grid size
- scrollback view changes
- plugin animation is active
- delayed render timers expire
- synchronized update timeout expires

This keeps idle usage low. The cursor trail is the main feature that requests
animation frames while no PTY data is arriving.

## Damage And Rendering

`c_term_core` records damage at the grid level:

- full viewport damage
- row/cell damage
- scroll damage
- cursor damage

`RenderCache` converts dirty rows into a CPU RGBA frame and tracks which rows
need uploading to the GPU texture. Scroll damage is optimized by copying rows
inside the texture and uploading only newly exposed rows.

The renderer presents a single terminal texture with shader-side cursor and
plugin overlays. Plugins do not mutate the terminal grid; they emit overlay
commands for the renderer.

## PTY Flow

Startup:

1. `c_term_app` opens a Wayland window.
2. It creates a PTY with the current grid size.
3. The child process sets terminal identity environment variables.
4. The child execs `$SHELL`.
5. A reader thread forwards PTY bytes into the event loop.

Input:

1. Keyboard and mouse events are encoded in `window_backend/input.rs`.
2. Bytes are written to the PTY master.
3. Terminal responses generated by `c_term_core` are also written to the PTY.

Resize:

1. Window size is converted to cell columns and rows.
2. PTY winsize is updated.
3. `TerminalCore` resizes its grids.
4. Render textures and caches are resized.

## Clipboard Flow

Normal app shortcuts:

- Ctrl-Shift-C copies Termite's visible text selection to the Wayland clipboard.
- Ctrl-Shift-V reads text from the Wayland clipboard and writes it to the PTY.
- Bracketed paste is respected when the application enables it.

OSC 52:

- `TerminalCore` parses OSC 52 clipboard writes.
- It emits a clipboard event in `CoreTick`.
- `c_term_app` decodes the base64 payload and writes it to the Wayland
  clipboard.

This is what makes tmux copy mode copy out of the terminal.

## Philosophy

Keep the terminal core small and boring. Put OS and GPU concerns in the app
crate. Keep visual extensions as plugins. Prefer explicit, compiled Rust config
over hidden runtime state.
